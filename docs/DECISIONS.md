- **Data Dragon as pinned snapshot** (not live fetch): version recorded in filename; staleness self-announces via the unmapped-champion crash.
- **Champion universe from Data Dragon, observations from CSV** (173 vs 172 — Locke).
- **Team identity: OE teamid as grouping key + surrogate keys + alias table**; cross-source knowledge lives in data/seeds/team_aliases.csv (data, not code — same philosophy as fearless_config).
- **Players: name-as-ID for v0** — surrogate keys without resolution knowledge would be a costume, not armor; player_aliases parked until evidence bites.
- **Admission policy: our fields, not OE's datacompleteness flag** — LPL kept (819/820 draft-complete); ~506 ID-less minor-league games + 1 contradiction game dropped-and-reported. Cleaner N over more N.
- **Crash vs skip doctrine: our bugs crash loudly; reality's holes get dropped-and-reported.**
- **Missed bans = NULL-champion action rows** (real game events, schema anticipated them).
- **Two-phase pipeline: compute/validate everything, then write** — a game_id in games is vouched for by the whole pipeline; stages never undo each other.
- **champion_meta / champion_role_meta as views** (derive-don't-duplicate; staleness impossible; soloQ-era may convert them to tables — accepted).
- **Schema applied by the ingest script, idempotently** (IF NOT EXISTS / OR REPLACE) — one-command rebuild from any state.
- **Parked**: minor-league rescue via name-minted teams; fearless_config must become seed-CSV-fed before 0.4b fills it.- **Patch ordering: numeric (major, minor) parse + LAG window, not VARCHAR sort** (padding trap: '15.2' > '15.10' lexicographically) **and not chronological** (region adoption jitter can misorder first-game dates). Crash on unparseable patch.
- **As-of layer as views over patch_order** (`meta_asof`, `synergy_asof`, `matchup_asof`): same-patch stats unreachable *through the views*. Caveat: convention-enforced for consumers — nothing physically stops a direct same-patch join on base tables; the rule is "meta features flow through *_asof, never champion_meta directly".
- **Relation "edges" stored as counts, not scores** (`pair_synergy_meta`/`pair_matchup_meta`): an edge (lift/score) is a Phase-3 feature; the counts beneath any conceivable edge definition are facts. Canonical order champ_a < champ_b (pairs stored once); matchup wins are from champ_a's arbitrary perspective (`n_wins_a`). Triples/quadruples rejected: ~0.23 obs/triple at pro scale — higher-order composition goes through archetypes, not enumeration.
- **INSERT BY NAME everywhere new** (ddragon first): positional `SELECT *` silently swaps columns when orders drift (attackspeed/attackspeedperlevel near-miss); BY NAME makes order meaningless and turns name mismatches into loud crashes.
- **Leaguepedia auth is mandatory** (bot-password via shared `lp_common.lp_login`): anonymous calls hit a far harsher rate limiter — this was the entire cause of a hostile afternoon. All LP scripts share `with_backoff` (exponential, no sleep before final surrender).
- **Tournament seed workflow: generate-then-curate** (`list_lp_tournaments.py` prints candidates, human prunes + fills oe_league) — automating the typing, never the judgment; name-pattern auto-selection rejected (substring thinking, silent both ways).
- **Tournament lookup via raw Cargo query, bypassing the wrapper**: `leaguepedia_parser.get_tournaments` cannot see the Asia Pacific region (joins Leagues + filters TournamentLevel); one raw-query code path for all regions instead of a special case.
- **Matcher is pure compute → derived CSV; series applied at insert time by ingest_oracle**: DuckDB implements UPDATE as delete+reinsert, so updating any FK-referenced games row throws. Bootstrap order on fresh clone: ingest → match → ingest.
- **Champion-set match key held at scale: 2,520/2,522 (spike's 15/15 vindicated).** Series boundary = gameInSeries reset to 1 in time-sorted tournament stream; `best_of = 2×(series winner's wins)−1`, underivable → drop-and-report.
- **fearless_config classified empirically from pick-repeat analysis, not public announcements**: all 2025 major tournaments = hard fearless, all year (0 repeats across 1,000+ multi-game series). Seed-CSV-fed, rebuild-proof.
- **Centerpiece operational definition sharpened: within 2025, "non-fearless training data" = game-1 + Bo1 states (undepleted availability); depleted states (games 2+) are the eval.** Cleaner than cross-tournament splits (same meta/teams, only availability differs). 2024 data = future literal-non-fearless option.
- **Fearless-unavailable-set helper lives in dataset.py, not the DB** (training-time computation over series state, not a stored fact).
